# 任务书

## 题目：非视距环境下视觉语言导航技术研究

### 内容及要求

长距离规划导航主要关注长席列或目标导向的无人机飞行指令，本项目的开发和实施中，要求模型具备高层规划动力学的低层运动的实时控制和长距推理能力。

### 主要指标

1. 多粒度导航历史收集模块:实现历史信息高效聚合；
2. 基于导航进程估计的语言-动作切换机制:统一动作建模和语言输出；
3. 全局地图提供广域环境感知，保障决策规划质量;
4. 无人机负责局域精细观察，确保飞行安全并搜寻目标

### 一些参考的要求

毕设开题报告的写法，请注意如下问题：
1. 文献综述/国内外研究现状一定要有参考文献，即里面综述的内容，都要是有来源的。参考文献列在综述的后方。
2. 参考文献注意格式，比较容易错的是姓名的大小写和次序，要求所有文献都统一（不要有的全大写有的全小写，不要有的姓前名后有的反过来）
   3.课题任务和要解决的问题用“总-分”结构写，总体任务可以照着任务书里面的，”要完成一个什么样的任务最终实现什么“。
   分任务可以自己总结两个点（比如用llm来预测金融市场，可以写（仅参考）：首先部署或接入LLM API，部署xxx框架，实现一个价值信息处理的框架；然后针对xx资产，进行xx预测分析），
   也可以照着任务书分3到4个点（个人不太推荐这样，显得太细碎，但是这样比较省事，看各人选择）
4. 在研究方案和拟采用方法里面，一定要切记，方案是针对上面提出的各个点的“任务”的。如果上面是分成了两个点，那么研究方案就是，针对研究问题一，拟采用xxx方法，然后把准备怎么做写得仔细一点，写一段话（最好有比较详细的方案，不一定正确，但是要显得经过了思考和推理，准备用什么方法一步一步实现）；针对研究问题二，blahblahblah。
   如果是分的4个点，那就是分别针对研究问题1、2、3、4来写，总之要和前文一一对应。这样读起来就非常有结构，开题时候评阅组很容易get到你要做什么，就比较容易过。
